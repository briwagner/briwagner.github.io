Refining the Site

November 24, 2015

I feel like we crossed the finish line, at least for phase 0 now with the completion of our personal sites. No doubt, there is more that could be accomplished, but it's good to see where I brought it as of today.

At the start, I certainly had notions for the site that didn't pan out. I imagined more biographical stuff, but what's the point of that? That's what myspace is for. Still I needed to complete the video section, and I added a couple neat vids, that provide a bit of my history for anyone interested.

More than anything, this final round of site edits needed to focus on refining the CSS and getting all the links to work. I finally tweaked the front page so the two main boxes will behave in a more orderly fashion. It should look the same as before, but work differently under the hood. I also took the time to isolate some of the CSS in a way is more flexible and easy to use going forward.

My one big challenge for myself with this was to figure out how to integrate some angular.js elements. And I got a handle on the basics of that framework. The two blog entries on the front page are all spit out from angular. And the blog index page is all angular as well. The blog sidebar too. It's a neat framework if you're looking for some auto-generated content abilities. I certainly didn't want to maintain all the menu links for the blog posts as they grew in number, so I went looking for a better way.

Basically, angular has a repeat function, where you design the layout once in the html. Add the appropriate angular commands and link to a js file with all the content in it, in the form of an object. Then it will spit out all the various elements from that js object, repeating the basic structure you created initially. So the html is this:

HTML markup for angular ng-repeat function

<tag> { blog.title } </tag>

I can't write all the formatting, because the page will interpret it! That should be a double curly braces to open and to close. So look at the source for sure. Anyway, I can put that in a few spots on the site, in different formats even. And put my updates in the js file once. Angular will read the full js object and ... repeat magic happens! It's kind of the poor man's CMS system. If the js file is like my database and the angular works as a controller for the html.
