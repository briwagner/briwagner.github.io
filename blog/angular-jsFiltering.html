Filtering for Angular JS

April 16, 2016

Angular has its quirks.

It's smart enough to flag duplicates in an array of data, but not smart enough to tell me what to do with that problem, for instance. So I turn to vanilla JS for a solution. And en route, I'll take a detour to recall Javascript idiosyncrasies, after spending the last couple of months entrenched in Ruby.

My first stop is forEach(). This certainly has been helpful in wrangling data from an API call into useful Angular bits. But it doesn't work for everything. Why? It won't break. When it says 'for each' it really means for each, and it won't break out of the loop early just because you find what you're looking for.

What other options are available? every() is one. It tests if ALL of the elements in an array pass the test and returns true or false. Kind of an all or nothing proposition.

var basket = ['orange', 'kiwi', 'banana'];
var citrus = ['lemon', 'orange', 'grapefruit'];

function isCitrus(val) {
  return citrus.includes(val);
}

var y = basket.every(isCitrus);

console.log(y)

=> false

some() is another. As you can guess, it's performing a similar check but only tells if you the condition is present at least once. Both of these functions return true/false.

var basket = ['orange', 'kiwi', 'banana'];

var citrus = ['lemon', 'orange', 'grapefruit'];

function isCitrus(val) {
  return citrus.includes(val);
}

var y = basket.some(isCitrus);

console.log(y)

=> true

This is where my Ruby months play tricks on my mind. Are every() and some() part of those additions made as part of ECMA 2015? Is indexOf the only reliable partner I have in testing for elements? What is this newfangled includes() method? It can't have much support, can it??

In my blind allegiance to the past, I eschewed what seemed new and searched for something old. I found it in the tried and true for loop. What more could I need? Well, quite a bit actually.
